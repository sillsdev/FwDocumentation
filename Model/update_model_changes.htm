Making model changes for the WW branch

Use the following guide to make sure all relevant changes are made.

·         Check out the following (You can use batch files to do this with Perforce):

·         C:\ww\Bin\src\XMI\ConceptualModelChanges.txt

·         C:\ww\Bin\src\XMI\FieldWorks.xml.zip

·         C:\ww\Doc\Database\ModelDocumentation.chm (generated Windows help file)

·         Unzip FieldWorks.xml.zip (C:\ww\Bin\src\XMI\FieldWorks.xml.zip).

·         Rename C:\fw\Bin\src\XMI\FieldWorks.xml (if it exists) to FW_FieldWorks.xml.

·         Rename C:\fw\Doc\Database\ModelDocumentation.chm to FW_ModelDocumentation.chm

·         Copy C:\ww\Bin\src\XMI\FieldWorks.xml to C:\fw\Bin\src\XMI\ FieldWorks.xml

·         Make sure to change the FWRoot and Path properties to FW!

·         Open FieldWorks.xml in MagicDraw. (C:\fw\Bin\src\XMI\FieldWorks.xml, the contents of the zip.)

·         Make changes to model using MagicDraw:

·         Be sure to include num tag on each new class and each new attribute. The first attribute for a new class starts at 1. (Make sure not to change existing numbers on existing attributes. For example, if you delete an attribute, do not reuse its number with any new attributes.) The generated list of current numbers is in C:\fw\Bin\src\XMI\Transforms\ClassList.htm. (Rerun all.bat to generate a new list)

·         Be sure to include abbrev tag on each new class.

·         Be sure to establish a superclass for any new classes (often CmObject). Explicity setting the base class is not sufficient; you must also drag the new class object node in "Containment Tree" Window under the appropriate namespace in "Component View".

·         Include any helpful documentation.

·         (First time or new version users: see C:\fw\Doc\Model\SetupModeling.htm)

·         Generate the HTML documentation from MagicDraw. (P 2.0 GHz takes about 3 minutes.) 

To do this, press the Generate button at the bottom of Tools\Report.

·         From the fw\bin\src\xmi\transforms directory, run CreateReplaceHelp.bat.  This file generates replacement files for all classes in the FW model and copies them to C:\fw\Bin\src\XMI\ReportCustomization\chm generation.

·         Copy the newly created files from C:\fw\Bin\src\XMI\ReportCustomization\MagicDraw UML Personal Edition\diagrams to C:\fw\Bin\src\XMI\ReportCustomization\chm generation/Diagrams.

·         From the C:\fw\Bin\src\XMI\ReportCustomization\chm generation directory, run CreateHelpFile.bat to create the new .chm documentation.

·         Move C:\fw\Bin\src\XMI\FieldWorks.xml to C:\ww\Bin\src\XMI\ FieldWorks.xml

·         Move C:\fw\Doc\Database\ModelDocumentation.chm to C:\ww\Doc\Database\ModelDocumentation.chm.

·         Rename  C:\fw\Doc\Database\FW_ModelDocumentation.chm back to ModelDocumentation.chm

·         Rename C:\fw\Bin\src\XMI\FW_FieldWorks.xml (if it exists) back to FieldWorks.xml

·         Put C:\ww\Bin\src\XMI FieldWorks.xml into the zip file (C:\ww\Bin\src\XMI\FieldWorks.xml.zip).

·         Make sure to change the FWRoot and Path properties back to WW!
